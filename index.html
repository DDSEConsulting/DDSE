What the Patent Covers

Data Driven Computing System Development

U.S. Patent 12,197,893 B2 describes a structured method for constructing computing system software based on a foundational concept: software behavior can be systematically defined through a limited set of data types and attributes.

Three Types of Data
The framework organizes system data into three primary categories.
Input
External Input Data Data received from sensors, communication interfaces, or user inputs.
Middle
Internal Data Stored and managed data including configuration, calibration, and intermediate computation results.
Output
External Output Data Data transmitted to actuators, displays, or external systems.
Two Systematic Attributes
Each data type is characterized by two key attributes.

Data Value

Represents the content or meaning of the data. Safety considerations include correctness, plausibility, and integrity.

Data Timing

Represents when data is available or processed. Safety considerations include timing constraints and deadlines.

This structure provides a consistent basis for analyzing software behavior in systems where correctness and timing are critical.
