BACKGROUND OF THE INVENTION
Field of the Invention
[0004] The invention discloses a definite approach to construct computing system software based on the data: Input data, Middle data and Output data, and the calculations or relationships between them and provides the safety mechanism development approach to make sure that the software will work correctly by focusing on only two unique systematic attributes: Data Value and Data Timing. In this way, the software constructions, the relevant software safety mechanisms developments and the measurement criteria for both software constructions and safety developments are defined once the data and the calculations between them are defined in the system. 

BRIEF SUMMARY OF THE INVENTION
The present invention provides a definite or fixed method for implementing a computing system software and safety mechanisms, and provides the clearly defined and specific unique criteria to measure both software constructions and the safety mechanisms for said software based on the data value and data timing attributes.

Safety

[00036] The disclosed embodiments describe the safety development in a computing system consisting of the reliability development, availability development, and quality control. 

Reliability: Ensures the software acts as implemented, detecting and reacting to errors in data value and data timing. Covers input, middle, and output data error detection, including measures like timeout monitoring, result plausibility checks, and redundant calculations.

Availability: Focuses on maintaining functionality through redundancy mechanisms (e.g., multiple object detection systems or braking subsystems).

Quality Control: Prevents deviations between requirements and implementations via verification, testing, and management processes. Emphasizes compliance with industry standards like ASPICE, ISO 26262, and UN ECE 155/156.

Claims

1. A method of constructing a computing system software on a computer system, wherein said method consists of:
   - Defining and implementing a plurality of data (input, middle, and output data) with two attributes: data value and data timing.
   - Developing safety mechanisms to detect and manage errors in data attributes (value and timing).

2. The method of claim 1, wherein each input, middle, and output data has defined protocols and timing attributes calculated based on transmission durations, processing times, and system overheads.

3. The method of claim 1, wherein safety mechanisms include error detection (value and timing), error reactions, and quality management.
